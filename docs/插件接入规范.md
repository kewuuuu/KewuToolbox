# KewuToolbox 插件接入规范（v1.0）

本文档定义插件向桌面端上报数据的统一协议。浏览器插件、IDE 插件（如 VSCode）以及其他应用插件都使用同一接口。

## 1. 目标

- 允许插件把“应用内更细粒度对象”上报给桌面端。
- 允许插件声明“需要抑制的原生进程窗口”，实现“插件对象替代原进程对象”。
- 统一数据结构，便于设置页展示“已连接插件列表”。

## 2. 上报地址

- 方法：`POST`
- 地址：`http://127.0.0.1:17321/plugin-bridge`
- 内容类型：`application/json`

说明：只支持本机回环地址，不对外网暴露。

## 3. 请求体结构

```json
{
  "protocolVersion": "1.0",
  "source": "browser-extension",
  "plugin": {
    "id": "official-browser-bridge",
    "name": "Kewu 浏览器桥接插件",
    "version": "0.3.0",
    "homepageUrl": "https://github.com/kewuuuu/KewuToolbox",
    "isOfficial": true
  },
  "snapshot": {
    "timestamp": "2026-04-21T10:00:00.000Z",
    "focusedClassificationKey": "plugin-browser-tab|https://example.com/path",
    "suppressRules": [
      {
        "namePattern": "*",
        "typePattern": "AppWindow",
        "processPattern": "msedge.exe"
      }
    ],
    "records": [
      {
        "classificationKey": "plugin-browser-tab|https://example.com/path",
        "displayName": "示例页面标题",
        "normalizedTitle": "https://example.com/path",
        "objectType": "BrowserTab",
        "processName": "msedge.exe",
        "domain": "example.com",
        "browserName": "Microsoft Edge"
      }
    ]
  }
}
```

## 4. 字段说明

### 4.1 顶层字段

- `protocolVersion`：协议版本，当前固定 `1.0`。
- `source`：插件来源标识，可选（示例：`browser-extension`、`vscode-plugin`）。
- `plugin`：插件元信息对象，必填。
- `snapshot`：本次快照数据，必填。

### 4.2 `plugin` 对象

- `id`：插件唯一 ID（必填，建议稳定不可变）。
- `name`：插件显示名（必填）。
- `version`：插件版本（必填）。
- `homepageUrl`：插件主页（可选）。
- `isOfficial`：是否官方插件（可选，布尔）。

### 4.3 `snapshot` 对象

- `timestamp`：快照时间（可选，ISO 字符串）。
- `focusedClassificationKey`：当前焦点记录键（可选）。
- `suppressRules`：抑制规则数组（可选）。
- `records`：插件记录数组（可选，推荐必填）。

## 5. 抑制规则 `suppressRules`

每条规则字段与黑名单匹配一致：

- `namePattern`：名称通配。
- `typePattern`：类型通配（如 `AppWindow`、`BrowserTab`、`Desktop`）。
- `processPattern`：进程名通配（如 `msedge.exe`）。

匹配行为：

- 同一规则中，填写的字段需同时匹配。
- 字段留空表示该字段不参与过滤。
- 命中抑制规则的桌面端原生对象将不再记录（由插件 `records` 替代）。

## 6. 插件记录 `records`

每条记录支持字段：

- `classificationKey`：记录唯一键（必填，建议稳定）。
- `displayName`：显示名称（必填）。
- `normalizedTitle`：标准化标题/标识（可选）。
- `objectType`：`AppWindow` / `BrowserTab` / `Desktop`（可选，默认 `AppWindow`）。
- `processName`：进程名（可选，默认 `plugin`）。
- `domain`：域名（可选）。
- `browserName`：浏览器名（可选）。

说明：

- `classificationKey` 决定聚合维度；同键会累计到同一对象。
- 若插件要“一个对象替代多个原始窗口”，让这些窗口命中 `suppressRules`，并让替代对象使用同一个 `classificationKey`。

## 7. 焦点同步

- 插件可通过 `focusedClassificationKey` 指定当前焦点插件对象。
- 当桌面端当前焦点窗口被 `suppressRules` 抑制时，桌面端将使用插件焦点对象参与焦点计时。

## 8. 心跳建议

- 推荐每 0.5 秒到 2 秒上报一次。
- 桌面端超过 90 秒未收到插件上报会自动判定为断开，并从“已连接插件”列表移除。

## 9. 兼容性

- 历史 `/browser-bridge` 接口仍保留兼容，不建议新插件使用。
- 新插件请统一接入 `/plugin-bridge`。
